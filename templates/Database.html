National Drug Overdose Deaths & Death Rates

The following cleaned CSV dataset was taken from the supporting Excel (.xlsx) data document for the National Institute of Drug Abuse's Overdose Death Rates report. The report's timeline spanned from 1999-2016.

The original Excel file had two separate Worksheets: National Overdose Deaths from Prescription & Illicit Drugs and National Overdose Death Rates. The worksheets were merged as one and extra columns that did not pertain to overall and gender specific information were dropped.

Next the existing Columns and Rows were transposed for X-Axis and Y-Axis placement so they may be easily visualized for the two Bar/Line Combo charts. The new Column headers were also reformatted so they could be easily read when converting into a SQLite database. Lastly, the cleaned Excel file was saved as a Comma Delimited (.csv) file.

The new CSV file was then imported into the SQLite database as the national table for the Opioid_OD schema. Initially, we used the sqlite3 tool for the importation. However, there were issues where the numeric cells were being listed as a TEXT data type. This was resolved by correcting the data types for the Death Rate columns as a FLOAT and the Death columns as a INTEGER using the DB Browser for SQLite tool. Also the year column was set as the Primary Key.

State Drug Overdose Deaths & Death Rates

The following cleaned CSV dataset was taken from the supporting CSV data documents for the Center for Disease Control and Prevention (CDC)'s Drug Overdose Mortality by State report. The report's timeline looked at the particular years: 1999, 2005, 2014, 2015 and 2016.

The original CSV files were five separate Workbooks. The workbooks were first merged as one and extra columns that did not pertain to deaths and death rates specific information were dropped. Also a new column named, ID, was added as the first column in the data file. This column would have unique TEXT type values added.

The new CSV file was then imported into the SQLite database as the state table for the Opioid_OD schema. Initially, we used the sqlite3 tool for the addition. However, there were issues where the numeric cells were being listed as a TEXT data type. This was resolved by correcting the data types for the Death Rate columns as a FLOAT and the Death columns as a INTEGER using the DB Browser for SQLite tool. Also the ID column was set as the Primary Key.
